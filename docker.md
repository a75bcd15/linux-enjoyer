# Docker
## Теория
1. **Dockerfile** - текстовый файл-инструкция, который описывает, как собрать образ (какая ОС, какие пакеты установить, какие файлы скопировать, какую команду запускать).
2. **Образ (Image)** - готовый неизменяемый шаблон, собранный из Dockerfile. Аналог `.iso` файла и хранится на диске.
3. **Контейнер (Container)** - запущенный экземпляр образа. Это изолированный процесс в системе. Контейнеров может быть много от одного образа.

*Dockerfile -> (build) -> Image -> (run) -> Container*

## Установка
1.  Установка пакета: `sudo pacman -S docker`
2.  Запуск и включение сервиса: `sudo systemctl enable --now docker`
3.  Настройка прав: `sudo usermod -aG docker $USER`
4.  Проверка:
```bash
docker --version
docker run hello-world
```

Docker хранит все данные (образы, контейнеры, volumes) в директории: `/var/lib/docker/`. 

## Команды
### Образы
- Скачать образ из Docker Hub:
```bash
docker pull <образ>:<тег>
```

- Удалить образ:
```bash
docker rmi <образ>:<тег>
```

- Собрать образ из Dockerfile:
```bash
docker build -t <имя_образа>:<тег> <путь_к_Dockerfile>
```

### Контейнеры
- Запустить контейнер из образа:
```bash
docker run -d --name <имя> <образ>
# -d: daemon mode (фоновый режим)
# --name: задать имя
```

- Посмотреть логи контейнера:
```bash
docker logs <имя_или_id>
docker logs -f <имя_или_id> # следить за логами в реальном времени
```

- Зайти внутрь работающего контейнера (bash):
```bash
docker exec -it <имя_или_id> /bin/bash
# или /bin/sh, если bash нет
```

- Остановить контейнер:
``` bash
docker stop <имя_или_id>
```

- Удалить контейнер:
```bash
docker rm <имя_или_id>
# docker rm -f <имя>
```

### Проверка состояния
- Список образов:
```bash
docker images
```

- Запущенные контейнеры:
```bash
docker ps
# docker ps -a - все контейнеры (включая остановленные)
```

### Очистка системы
- Удалить всё неиспользуемое (остановленные контейнеры, пустые сети):
```bash
docker system prune
```

- Удалить всё, включая неиспользуемые образы и volumes
```bash
docker system prune -a --volumes
```
