# IP
Internet Protocol или **Межсетевой протокол** позволяет взаимодействовать узлам, находящимся в разных локальных сетях. Существует 2 версии: [**IPv4**](ipv4.md) и [**IPv6**](ipv6.md). Сравнить их можно с помощью [**этого файла pcapng**](../etc/http_ip4and6.pcapng) через [**WireShark**](wireshark.md).

Узнать свой IP-адрес:
- `ipconfig` в Windows;
- `ifconfig`, `ip addr` в Linux.
# Время жизни пакета, TTL
Величина **времени жизни**, или **Тimе to Live - TTL**, определяет период времени, которое может истечь, или максимальное количество маршрутизаторов, которые может пройти пакет, прежде чем он будет аннулирован. Мера необходима для предотвращения бесконечно перемещающихся пакетов, которые могут замедлить работу сети.
# Подсети и CIDR

# local host
**Localhost** (также известный как **loopback address**) - термин, используемый для определения сетевого адреса, который используется устройством для обозначения самого себя. Способ устройства установить сетевое подключение к самому себе. `127.0.0.1` зарезервирован в качестве адреса обратной связи в сетях IPv4, для сетей IPv6 он представлен в виде `::1`.
