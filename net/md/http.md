# HTTP
**Hypertext Transfer Protocol** или **Протокол передачи гипертекста** служит в качестве механизма доставки содержимого, давая веб-браузерам возможность подключаться к веб-серверам для просмотра веб-страниц. Существует множество реализаций заголовка пакета, при этом в них могут присутствовать основные **методы**.

**Получение данных от сервера.** Например, для того, чтобы просмотреть веб-страницу, клиент посылает запрос для загрузки (по **методу** **`GET`**) корневого каталога `/`  (в виде **URI** (Uniform Resource Indicator - универсальный указатель ресурсов)) на веб-сервере. Так как НТТР использует [**ТСР**](tcp.md) для передачи, то в WireShark после НТТР-запросов последуют ТСР-пакеты, в которых нет заголовков НТТР.

**Публикация данных.** Например, пользователь публикует  комментарий на веб-сайте. После трехэтапного рукопожатия TCP от клиента идёт пакет с **методом `POST`**, в котором указывается URI (/page.php). Далее сервер переадресовывает клиента, чтобы тот снова отправил запрос по методу `GET` (обновил страницу). Прочитать такой комментарий можно будет во вкладке `HTML Form URL Encoded: application/x-www-form-urlencoded` в WireShark.
## Извлечение данных
Так как трафик по HTTP не зашифрован, то с помощью [**WireShark**](wireshark.md) можно просматривать введённые пароли пользователем или извлекать переданные изображения. На [**этом сайте**](https://wiki.wireshark.org/SampleCaptures) представлены различные файлы перехвата.

Взяв тот, что содержит JPEG картинки, можно найти в нём пакет содержащий изображения (в описании будет указано `OK` и, например, `JPEG JFIF image`). Нажав правой кнопкой мыши по вкладке `JPEG File Interchange Format`, нужно выбрать пункт `Export Packet Bytes...` и сохранить картинку с указанием формата JPEG.
