# Профили конфигурации
В профиле конфигурации хранится следующее:
- Глобальные параметры настройки;
- Фильтры перехвата;
- Фильтры отображения;
- Правила выделения цветом;
- Запрещенные сетевые протоколы;
- Принудительные расшифровки;
- Недавние установки, в том числе размеры панелей, настройки представления меню и ширина столбцов;
- Характерные для протоколов таблицы, содержащие, например, перечень пользователей сетевого протокола SNMP и специальные НТТР-заголовки.

Чтобы просмотреть список профилей конфигурации, выберите команду **Edit > Configuration > Profiles** из главного меню. Можно также щелкнуть правой кнопкой мыши на разделе профилей в правом нижнем углу окна и выбрать команду Manage Profiles из контекстного меню.
# Фильтры
## Фильтры перехвата
**Синтаксис BPF** (Berkeley Packet Filter - фильтр пакетов Беркли) характерен для целого ряда приложений анализа пакетов. Фильтр, созданный с помощью синтаксиса BPF, называется **выражением**, при чем каждое выражение состоит из одного или нескольких **примитивов**. В свою очередь, примитивы состоят из одного или нескольких **квалификаторов**:

| Квалификатор | Описание | Примеры |
| :--- | :--- | :--- |
| type | Обозначает имя или номер идентификатора, на который делается ссылка | host, net, port |
| dir | Обозначает направление передачи при обмене данными с сетевым узлом | src, dst |
| proto | Ограничивает соответствие конкретному протоколу | ether, ip, tcp, udp, http, ftp |

Операции: 
- И (&&);
- ИЛИ ( || );
- НЕ (!).

**Примеры:** 
- `tcp [13] 18` Пакеты ТСР с установленными флагами SYN и АСК
- `tcp [13] & 16 == 16` Пакеты ТСР с установленным флагом АCK
- `ether host 00:00:00:00:00:00` Входящий и исходящий сетевой трафик по указанному МА С-адресу
- `broadcast` Только широковещательный трафик
- `ip` Трафик только по сетевому протоколу 1Pv4
## Фильтры отображения
| Операция | Описание |
| :--- | :--- |
| == | Равно |
| != | Не равно |
| > | Больше |
| < | Меньше |
| >= | Больше или равно |
| <= | Меньше или равно |
| and | Оба задаваемых условия должны быть истинны |
| or | Истинным должно быть хотя бы одно задаваемое условие |
| xor | Истинным должно быть одно и только одно условие |
| not | Ни одно из задаваемых условий не истинно |

**Примеры:** 
- `!tcp.port 3389` Отсеять сетевой трафик по протоколу RDP
- `tcp.flags.syn=1` Отобразить пакеты ТСР с установленным флагом SYN
- `!arp` Удалить сетевой трафик по протоколу ARP
- `http` Отобразить весь сетевой трафик по протоколу НТТР
- `tcp.port=23 || tcp.port=21` Отобразить сетевой трафик по протоколу Telnet или FTP
# Статистика и преобразование
## Узлы и диалоги между ними
Узнать конченые точки (узлы): **Statistics > Endpoints**
- Name resolution (Преобразование имен)
- Limit to display filter будут отображаться только те точки, которые удовлетворяют фильтру отображения

Узнать диалоги между конечными точками (узлами): **Statistics > Conversations**
- A - отправитель, B - получатель

Устройство, вызвавшее самый интенсивный обмен данными в перехваченном трафике, называется наиболее активным сетевым узлом (**top talker**). С помощью WHOIS-сервиса можно узнать, кому принадлежит адрес, если он внешний (межсетевой).
## Иерархия сетевых протоколов
**Statistics > Protocol Hierarchy**
- В процентах можно увидеть, какая доля трафика приходится на каждый протокол.
## Преобразование имен
**Edit > Preferences > Name Resolution**
- Поставить галочки на MAC addresses, transport names, network addresses.

Либо можно нажать на отдельный пакет правой кнопкой мыши > **Edit Resolved Name** и указать нужное нам имя (оно не сохранится после закрытия программы).
## Дешифрование сетевых протоколов
Правой кнопкой мыши по пакету > Decode As > составить новое правило, например, 
`TCP port 443 Integer, base 10 TLS, FTP`.
## Отслеживание потоков
Выписка из пакетов сортирует данные с целью упростить просмотр пакетов от клиента к серверу.
- Поток TCP (http, ftp);
- Поток UDP (dns);
- Поток SSL (где шифруется);
- Поток HTTP.

Правой кнопкой мыши по пакету > Follow > ... Stream
- красным текст - от отправителя к получателю;
- синим - сетевой трафик от получателя к отправителю.
## Длина пакетов
**Statistics > Packet Lengths** 
- Обычный размер фрейма в сети Ethernet 1518 ( без заголовка 1460 байт). Если больше половины пакетов имеют длину пакетов выше 1280, то можно предположить, что в такой сети передается большой объем данных. Если меньше, то перехваченный трафик состоит из команд управления сетевыми протоколами.
## Составление графиков
**Statistics > IO Graph**
- Время круговой передачи (Round-Trip Time, RTT) - время, которое требуется на получение подтверждения доставки пакета адресату.

**Statistics >TCP Stream Graphs > Round Trip Time**
- По оси Y можно выявить большие величины времени задержки (с точки зрения пропускной способности сети)

**Statistics > Flow Graph**
- График потоков наглядно демонстрирует запросы и ответы. полезен и для понимания обычного потока данных.
## Экспертная информация
**Analyze > Expert lnformation**

Cостояния в пакетах протокола:
1. Chat (Текстовый диалог); 
2. Note (Уведомление);
3. Warning (Предупреждение); 
4. Error (Ошибка).
