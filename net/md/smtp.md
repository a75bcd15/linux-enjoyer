# SMTP 
**Simple Mail Tгansfer Protocol** или **Простой протокол передачи электронной почты**. Название говорит само за себя.

Рассмотрим пример экосистемы Microsoft. **Почтовый агент пользователя (Mail User Agent - MUA)**, представляющего собой клиентскую программу для работы с электронной почтой (Microsoft Outlook), посылает сообщение **агенту пересылки почты (Mail Transfer Agent - МТА)** - почтовому серверу (Microsoft Exchange). Если электронное сообщение предназначается для того же самого домена, откуда она отправляется, то агент МТА может непосредственно доставить ее получателю. Если же электронная почта посылается в другой домен, агент МТА обращается к DNS-серверу, чтобы найти IР-адрес почтового сервера получателя, а затем передать ему сообщение. Почтовый сервер нередко состоит из других компонентов, наподобие **агента по доставке почты (Mail Delivery Agent - MDA)** или **агента отправки почты (Mail Submission Agent - MSA)**. 

В протоколе SMTP используется простой набор команд и значений их параметров, посылаемых клиентом, после чего следует код ответа от сервера, что напоминает работу НТТР и TELNET. Команда `МAIL` с параметром `FROM:` - обозначает отправителя, а команда `RCPT` с параметром `ТО:` - получателя. Команда  `DATA` содержит само сообщение, наконец команда `QUIT` указывает на завершение передачи и происходит корректное завершение по протоколу TCP.

SMTP передаёт и текст, и вложения, указывая серверу, что передаются разнотипные данные со своим особым типом MIME и кодировкой.
## Этапы пересылки
1. **От клиента к локальному серверу.** 
	- Клиент устанавливает связь с локальным сервером электронной почты через TCP;
	- Сервер отправляет SMTP-запрос, объявляя свою версию ОС и какой протокол поддерживает (например, ESMTP - расширенную спецификацию SMTP) и пр;
	- Клиент отвечает командой `EHLO` (`HELO`, если ESMTP не доступен), предназначенной для идентификации хоста отправителя;
	- После сервер отвечает списком команд, которые поддерживаются SМТР-сервером, и какими командами разрешается пользоваться при передаче сообщения;
	- Тогда клиент посылает электронное письмо.
2. **От локального к удалённому серверу.** 
	Теперь локальный сервер выступает в качестве клиента по отношению к удалённому серверу и весь первый этап повторяется. Сервер электронной почты обнаруживает другой почтовый сервер, используя специальный тип записи **DNS МХ (mail exchange)**, ответственной за обмен почтой.
	
	В действительности на локальном и удаленном серверах электронной почты могут и не поддерживаться одни и те же функциональные средства, а возможно, эти серверы базируются на совершенно разных платформах. Именно поэтому так важен первоначальный обмен данными по протоколу SMTP. Ведь он позволяет серверу получателя передать отправителю поддерживаемый на нем перечень функциональных средств.
3. **От к удалённого сервера к удалённому клиенту.**
	Снова устанавливается надёжное соединение между сервером и клиентом. После извлечение электронной почты происходит с помощью других протоколов, например, **РОР3** (Post Office Protocol version 3 - Почтовый Протокол версии 3) и **IMAP** (Internet Message Access Рrotосоl - Протокол доступа к сообщениям в Интернете). Команда `STARTTLS` говорит о том, что сетевой трафик по протоколу IМАР будет зашифрован по протоколу **TLS** (Transport Layer Security - Протокол защиты транспортного уровня).
