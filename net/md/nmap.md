# Network Mapper
[**Официальный сайт**](https://nmap.org)
# Поиск хостов
`nmap -sn [IP-адреса / Диапазон]` Без сканирования портов (No Port Scan)
- Сначала Nmap отправляет эхо-запрос ICMP,
- далее запрос метки времени ICMP,
- пакет ACK на 80 порт (HTTP) и пакет SYN - на 443 (HTTPS),
- а после запрос ARP.
# Поиск открытых портов -p
`nmap -sS [IP-адреса / Диапазон]` SYN-сканирование портов TCP (скрытое сканирование);
- SYN-запрос на сервер и, получив в ответ SYN/ACK, покажет, что порт открыт. Если же сканер не получил SYN/ACK, то порт либо закрыт, либо фильтруется (защищает сетевой экран).

`nmap -sU -T5 [IP-адреса / Диапазон]` сканирование портов UDP;
- Сканер отправит UDP-пакет. Если Nmap получил ответ или ответа не последовало, то, возможно, порт открыт. Если сканер получил ошибку ICMP, то порт либо закрыт, либо фильтруется.
- UDP работает медленно из-за того, что не требует установления соединения. Для этого используется **параметр интервалов времени** `T5`, чтобы ускорить сканирование. Брандмауэры могут блокировать сканирование с T5 для интернет-адреса, можно использовать T2.

`nmap -p № [IP-адреса / Диапазон]` сканирование определенного порта;

`nmap -p- -T5 [IP-адреса / Диапазон]` сканирование всех портов.
# Поиск версий сервисов -sV
`nmap -p- -T5 -sV [IP-адреса / Диапазон]` 
- Сканирование версии займет больше времени, чем обычное сканирование портов, поскольку сканер будет пытаться определить тип сервиса. Чтобы увеличить скорость, лучше указать конкретные порты.
# Поиск ОС -O
`nmap -sV -O -T4 [IP-адреса / Диапазон]`
# Nmap Scripting Engine, NSE
Механизм сценариев Nmap cодержит набор дополнительных функций (таких, как перебор методом грубой силы, перебор DNS, перебор HTTP и т д ). Готовые сценарии хранятся в `/usr/share/nmap/scripts/`. Стоит обратить внимание, что некоторые сценарии требуют дополнительных аргументов. Ознакомиться можно в [**справочнике**](https://nmap.org/nsedoc/). Группы сценариев:
- Auth — аутентификация;
- Broadcast — широковещание;
- Default — по умолчанию;
- Discovery — обнаружение;
- DOS — отказ в обслуживании;
- Exploit — эксплойт;
- External — внешние;
- Fuzzer — фаззер;
- Intrusive — вторжение;
- Malware — вредоносные программы;
- Safe — безопасные;
- Version — версии;
- Vuln — уязвимости.

`nmap -p [номер порта] -sV –script [сценарий NSE] [IP-адрес / Диапазон]`

`nmap -sV -sC [IP-адрес / Диапазон]` сканирование сценариями по умолчанию 

`nmap -A [IP-адрес / Диапазон]` включает в себя:
- сканирование версий;
- сканирование SYN;
- сканирование сценариями NSE по умолчанию;
- сканирование операционной системы;
- трассировку по сети.

Сканирование несколькими сценариями в одной категории также осуществляется с помощью `*` (wildcard). Например:

`nmap -sV -p 135,445 --script smb-vuln* 10.0.0.187`
