# Анализатор пакетов
**Анализ пакетов** (протоколов) описывает процесс перехвата и интерпретации действующих данных по мере их продвижения по сети. Как правило, анализ пакетов проводится **анализатором пакетов (сниффером)** - инструментальным средством, применяемым для перехвата первичных данных, передаваемых по проводам сети. 

Принцип действия анализаторов пакетов:
1. Сбор двоичных данных из сети через смешанный режим (promiscuous mode) в сетевом интерфейсе (в этом режиме принимается весь трафик в сегменте сети, а не только адресуемый). 
2. Двоичные данные преобразуются в удобочитаемую форму.
3. Анализатор пакетов проводит анализ перехваченных и преобразованных данных. В частности, он проверяет протокол перехваченных в сети данных, исходя из извлекаемой информации, и далее начинает анализ характерных особенностей этого протокола.

Известные анализаторы пакетов:
- [**tcpdump**](tcpdump-tshark.md);
- [**WireShark**](wireshark.md).
# Список инструментов для анализа пакетов
- [**CloudShark**](https://www.qacafe.com/analysis-tools/cloudshark/) - решение для хранения, индексирования и сортировки перехваченных пакетов. 
- [**WireEdit**](https://omnipacket.com/wireedit) позволяет редактировать значения в отдельных полях пакета. Нужная вещь, чтобы протестировать применяемую систему обнаружения вторжений или разработанное сетевое программное обеспечение.
- Cain & Abel - устаревшая программа, используемая для заражения АRР-кеша.
- [**Scapy**](https://scapy.net/) - библиотека на языке Pythoп, с помощью которой можно создавать пакеты и манипулировать ими по сценариям из командной строки.
- [**TraceWrangler**](https://www.tracewrangler.com/) делает в перехваченных пакетах более анонимными различные типы присутствующих в них адресов. 
- [**Tcpreplay**](https://tcpreplay.appneta.com/) осуществляет повторную передачу, чтобы посмотреть, как на них будут реагировать сетевые устройства.
- [**NetworkMiner**](https://www.netresec.com/?page=NetworkMiner) применяется для судебной экспертизы сетей и включает, например, синтаксический анализ пакетов, извлечение переданных файлов.
- [**CapTipper**](https://github.com/omriher/CapTipper) предназначено для специалистов по безопасности сетей, занимающихся анализом сетевого трафика по протоколу НТТР, а именно поиск переадресации, файловых объектов и зловредного содержимого.
- [**ngrep**](https://github.com/jpr5/ngrep) позволяет находить конкретные данные в перехваченных пакетах.
- [**hping**](https://www.hping.org/wiki-sub/94/) служит для обработки, правки и передачи пакетов в режиме командной строки.
