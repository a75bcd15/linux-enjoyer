# Bash
## Упомянутые команды
- `echo`, `printf` вывод текста
- `read` чтение с клавиатуры

## Переменные
Существуют два типа переменных:
- Переменные среды (системные, например, $HOME);
- Пользовательские переменные (собственные переменные хранят значение до тех пор, пока не завершится выполнение сценария).

Символы описывают, по какому шаблону выдать содержимое (пример ниже):
%s — строка (текст);
%d — десятичное число;
%f — число с плавающей точкой;
%x — шестнадцатеричное число;
\n — новая строка;
\r — возврат каретки;
\t — горизонтальный отступ (табуляция).

В других строках этого файла символ решётки используется для обозначения комментариев, которые оболочка не обрабатывает. Однако, первая строка называют **шебангом**, указывающем системе на то, что сценарий создан именно для оболочки bash.

Переменные вставляются в строку с помощью знака доллара:
```bash
#!/bin/bash
SEX="WOMAN"
printf "I AM $SEX\n"
```
В переменную можно вложить вывод команды:
```bash
# альтернатива mydir=`pwd`
SORTED=$(cat months | sort)
printf "$SORTED\n"
```
При вызове скрипта указываются параметры (`./myscript.sh 2 3`), а действие над переменными записывается в двойных скобках :
``` bash
NUM1=$1
NUM2=$2
TOTAL=$(($NUM1+$NUM2))
printf "%s %d\n" "The total is " $TOTAL
```
Можно указывать значения по умолчанию, если не указан параметр пользователем:
``` bash
NUM1=${1:-0}
NUM2=${2:-0}
```
Узнать количество, введенных параметров можно с помощью `$#`.
Чтение с клавиатуры:
```
read -p "Your first name:" FIRSTNAME
read -p "Your last name:" LASTNAME
printf "Your full name is $FIRSTNAME $LASTNAME\n"
```

В некоторых ситуациях требуется экранирование символов. Поможет использование управляющего символа, обратной косой черты, перед  знаком доллара: `echo "I have \$1 in my pocket"`. 

## Функции, условия и циклы
Функции записываются в подобной форме и вызываются в конце:
```bash
NUM1=${1:-0}
NUM2=${2:-0}
function total(){
TOTAL=$(($NUM1 + $NUM2))
}
function print_answ(){
printf "%s %d\n" "The total is " $TOTAL
}
total
print_answ

```
Структура условия:
``` bash
if [[ сравнение ]]
then
something
else
something
fi
```

В скриптах можно **сравнивать числовые значения**:
- n1 -eq n2 Возвращает истинное значение, если n1 равно n2.
- n1 -ge n2 Возвращает истинное значение, если n1 больше или равно n2.
- n1 -gt n2 Возвращает истинное значение, если n1 больше n2.
- n1 -le n2 Возвращает истинное значение, если n1 меньше или равно n2.
- n1 -lt n2 Возвращает истинное значение, если n1 меньше n2.
- n1 -ne n2 Возвращает истинное значение, если n1 не равно n2.

**Сравнение строковых значений:**
- str1 = str2 Проверяет строки на равенство, возвращает истину, если строки идентичны.
- str1 != str2 Возвращает истину, если строки не идентичны.
- str1 < str2 Возвращает истину, если str1 меньше, чем str2.
- str1 > str2 Возвращает истину, если str1 больше, чем str2.
- -n str1 Возвращает истину, если длина str1 больше нуля.
- -z str1 Возвращает истину, если длина str1 равна нулю.

В командах сравнения прописные буквы меньше строчных. Сравнение строк здесь выполняется путём сравнения ASCII-кодов символов, порядок сортировки, таким образом, зависит от кодов символов.

**Сравнение файлов:**
- -d file Проверяет, существует ли файл, и является ли он директорией.
- -e file Проверяет, существует ли файл.
- -f file Проверяет, существует ли файл, и является ли он файлом.
- -r file Проверяет, существует ли файл, и доступен ли он для чтения.
- -s file Проверяет, существует ли файл, и не является ли он пустым.
- -w file Проверяет, существует ли файл, и доступен ли он для записи.
- -x file Проверяет, существует ли файл, и является ли он исполняемым.
- file1 -nt file2 Проверяет, новее ли file1, чем file2.
- file1 -ot file2 Проверяет, старше ли file1, чем file2.
- -O file Проверяет, существует ли файл, и является ли его владельцем текущий пользователь.
- -G file Проверяет, существует ли файл, и соответствует ли его идентификатор группы идентификатору группы текущего пользователя.

**Структура циклов while и for:**
```bash
while [[ условие ]]
do
что-нибудь
done

for i in {Список элементов}
do
что-нибудь
done
```
Итерация по файлу выглядит так:
```bash
for line in $(cat filename)
do
something
done
```
**Задача:** перенести содержимое всех файлов с расширением .md в рабочем каталоге в один текстовый файл так, чтобы сначала было название файла, а потом содержимое. **Решение:**
``` bash
#!/bin/bash
for file in ./**.md;
do 
echo "${file##/}"
cat $file 
echo " "
echo " "
done

# ./scrpt.sh > result.txt
```

## Регулярные выражения
Средство указания шаблона для поиска его в тексте. Хорошо работают в сочетании с grep командой.
- ^ - начало строки.
- $ - конец строки.
- `[]` - любой символ из заключённых в скобки. Поддерживает диапазоны, например [0-9] - цифры, [a-zA-Z] - все буквы латинского алфавита.
- `[^]` - любой символ за исключением заключенных в скобки.
- `\` - отменяет действие любого метасимвола. Например `\\$` - обозначает символ `$`, а не `\` в конце строки, а `\\$` - символ `\` в конце строки.
- `.` - любой один символ.
- `*` - 0 или более раз в тексте встречается предыдущий шаблон. Так например выражение `.*` означает любой набор символов.
