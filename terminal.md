При загрузке системы сначала запускается ядро (процесс 0), которое в свою очередь запускает командный интерпретатор **shell** (процесс 1). Взаимодействие пользователя с системой происходит в интерактивном режиме посредством командного языка. Оболочка операционной системы – shell – интерпретирует вводимые команды, запускает соответствующие программы (процессы), формирует и выводит ответные сообщения. Shell не только интерпретирует команды, но и создает среду, которую можно конфигурировать и программировать. У shell есть свой язык программирования, который позволяет писать программы, содержащие достаточно сложные последовательности команд.

Каждому пользователю предоставляется свой собственный пользовательский интерфейс, или shell. Строка, в которой набирается команда, называется командной строкой.

# Назначение Tmux
`tmux` откроет новый сеанс, что:
- позволяет запускать несколько терминальных сеансов в одном окне;
- помогает сохранить сессии, если соединение разорвалось;
- позволяет совместно с другими работать в терминале и отображает, что делает каждый пользователь.

# Команды Linux 
## Общие сведения
- ключ `-h` или `--help` : справка по любой команде
- `man`: справка по командам
- `whatis <название_команды>` : ищет заголовки man-страниц по целым словам (Пример: `whatis pwd`)
- `apropos <название_команды>` : ищет заголовки man-страниц по образцу, ищет все упоминаемые символы (Пример: `apropos ls`)
- `sudo` : для выполнения системных команд пользователями с низким уровнем привилегий (входящих в группу sudo)
- `history` список недавно выполненных команд
- символ `|` : канал команд (пайп), объединить вывод одной команды и отправить его следующей

## Системные
- `reboot` перезапустить ОС
- `poweroff` выключить ОС
- `fdisk` показать устройства, подключенные к ОС
- `lsblk` отобразить список дисков
- `lsusb` отобразить список USB-устройств
- `mount /dev/sdb1 /mnt/usb` примонтировать USB-накопитель
- `umount /mnt/usb`  извлечь USB-накопитель
- `uname` отобразить информацию об ОС

## Сетевые
- `hostname` отобразить имя хоста
- `ifconfig`, `ip addr` информация о сетевом интерфейсе
- `netstat -antu` информация о портах (ключи означают: показать все сокеты, не разрешать имена, показать TCP и UDP-соединения)

## Управление пакетами
- `apt update && apt upgrade -y` : `update` обновляет список пакетов последними версиями, `upgrade` обновит и установит последнюю версию пакетов, которые уже были получены (с помощью команды `update`)
- `full-upgrade -y` полностью обновить одну версию до другой
- `dpkg -l` вывести список всех установленных пакетов ПО
- `apt install [имя пакета] -y` установка нового ПО
- `dpkg -i [filename.deb]` установка ПО с расширением .deb
-  `apt remove [имя пакета]` удалить существующее приложение
- `apt-cache search keyword [ключевое слово для поиска]` поиск пакетов репозитория
- `apt-cache show [имя программы]` просмотр информации

## Управление сервисами и процессами
Сервисы — это серверы, которые могут работать в ОС, например, SSH, Web, FTP.
- `service [имя сервиса]` status получить статус сервиса
- `systemctl status [имя сервиса]`
- `service [имя сервиса] start` запустить сервис
- `systemctl start [имя сервиса]`
- `service [имя сервиса] stop` остановить сервис
- `systemctl stop [имя сервиса] `
- `service [имя сервиса] restart` перезапустить сервис
- `systemctl restart [имя сервиса] `
- `systemctl enable [имя сервиса] `включить автоматический запуск сервиса при старте системы
- `systemctl disable [имя сервиса]` отключить автоматический запуск
- `htop` отображение всех запущенных процессов
- `ps -au` отображение всех запущенных процессов
- `pgrep <опции> <процесс>` : показывает все PID определенного процесса (Пример: `pgrep -l sshd`)
- `fuser` : идентификация процессов, используя файлы или сокеты. Примеры:
  - `fuser -va 22/tcp` покажет PID процесса, использующий 22 порт; 
  - `fuser -va /home` покажет PID процесса, использующий домашний каталог; 
  - `fuser -k -TERM -m /home` завершит все процессы, имеющие доступ к домашнему каталогу.
- `lsof <название_файла>` : покажет список процессов, использующих указанный файл или каталог (Пример: `lsof /home`)
- `kill [PID]` завершить процесс
- `kill -9 [PID]` завершить процесс принудительно
- `killall` : убивает (завершает) процессы по имени. (Пример: `killall 0 httpd`)

## Пользователи и группы
- `useradd -m [имя пользователя] -G [имя группы] -s [тип командной оболочки]` добавить нового пользователя (эту же информацию можно просмотреть в системном файле `/etc/passwd`)
- `passwd  [имя пользователя]` задать пароль 
- `su [имя пользователя]` переключиться на другого пользователя
- `userdel [имя пользователя]` удалить пользователя 
- `sudo -l`  узнать права текущего пользователя
- `id` информация о текущем пользователе
- `w` или `who` (с меньшими подробностями) вывести список пользователей, которые в настоящее время вошли в систему
- `whoami` : выводит имя пользователя, который задействован в данный момент
- `last` вывести список последних авторизованных пользователей
- `groupadd [имя новой группы]` создать новую группу
- `usermod -aG [имя группы] [имя пользователя]` добавить пользователя в группу
- `groups <имя_пользователя>` выводит список групп, в которых состоит пользователь

## Права доступа файлов и каталогов
- `chmod [число, обозначающее права доступа] [имя файла]` выдать права доступа

Права доступа разделены на три категории:
- чтение (r): 4;
- запись (w): 2;
- исполнение (x): 1.

Для установки прав доступа применяется следующий шаблон: `[Пользователь:r/w/x] [Группа:r/w/x] [Все:r/w/x]`

Субъекты: 
- «u» - владелец,
- «g» - группа файла,
- «o» - все остальные пользователи.

Знак: «+» - добавить права, «-» - убрать права.

Пример: `chmod o+x /home/file` (всем остальным пользователям разрешить открывать файл).

Предусмотрены числовые режимы:
- 7: rwx,
- 6: rw-,
- 5: r-x, 
- 4: r--, 
- 3: -wx, 
- 2: -w-, 
- 1: --x, 
- 0: ---

Пример: `chmod 710 /home/file`, т.е. -rwx--x--- все права у владельца и исполнение у группы.

Для того, чтобы задать бит suid введите `chmod u+s <название/путь_файла>`, для бит sgid введите `chmod g+s <название/путь_файла>`. Для установки suid, sgid и sticky используется четырехзначный режим прав доступа.

| suid | sgid | sticky | режим |
| :--- | :--- | :--- | :--- |
| on | on | on | 7 |
| on | on | off | 6 |
| on | off | on | 5 |
| on | off | off | 4 |
| off | on | on | 3 |
| off | on | off | 2 |
| off | off | on | 1 |
| off | off | off | 0 |

Пример: `chmod 4775 /home/file`. Вывод команды `ls -l`: `-rwsrwxr-x 1 user user 0 Sep  9 12:42 /home/file`.

- `chown` : изменяет владельца или группу файла. Синтаксис: `chown <имя_пользователя/группы> <название_файла>`
- `chgrp <имя_пользователя/группы> <название_файла>` : изменяет группу файла
- `umask <значение>` : когда процесс создает новый файл, он указывает права доступа (зачастую 0666 — чтение и запись всеми). При этом система обращается к параметру **umask**, который задает, какие права доступа **должны быть отключены**. Синтаксис: `umask` (Пример вывода: `0002`). Изменение значения: `umask 0077`.

## Файлы и каталоги
- `ls -la [путь к целевой папке]` вывести список файлов и подкаталогов внутри любого каталога (`-a` : все содержимое, включая скрытые файлы; `-l` : форматированный список; `-X` сортирует содержимое по алфавиту)
- `touch` создать пустой файл
- `file [имя файла]` узнать тип файла в системе Linux
- `cp [путь к исходному файлу] [путь файла назначения]` скопировать файл
- `mv [путь к исходному файлу] [путь файла назначения]` переместить (вырезать) файл или переименовать
- `rm [путь к файлу]` удалить файл
- `rm -r` для рекурсивного удаления целого каталога с его внутренними файлами
- `pwd` вывести текущий рабочий каталог
- `cd [путь до нового каталога]` изменить текущий рабочий каталог. Если ввести `cd` без указания пути, пользователь попадёт в домашний каталог; если `cd ..` — перейдёт в предыдущий каталог.
- `mkdir [имя нового каталога]` создать каталог
- `rmdir <опции> <название>` : удаляет указанный пустой каталог
- `cp -r`, `mv -r`, `rm -r` для работы с каталогом и его внутренними файлами
- `wget [url]`  скачивание файлов из интернета/локальной сети
- `curl -O [url]` скачивание файлов
- `du <опции> <название_файла>` : отображает размер указанного файла или каталога
- `ln` : создаёт ссылки между файлами. 
  - Синтаксис жёсткой ссылки: `ln -P <название_файла> <название_ссылки>`
  - Синтаксис символической ссылки: `ln -s <название_файла> <название_ссылки>` (Пример: `ln -s file.txt soft_link`)
- `lpr <название_файла>` отправить файл на печать
- `lpq` проверить ход выполнения печати

## Текстовые файлы
- `echo` печать текста в терминал
- `cat` отображение содержимого текстового файла
  - `-b` — пронумеровать все непустые строки;
  - `-n` — пронумеровать все строки; 
  - `-s` — отобразить несколько подряд идущих пустых строк в виде одной; 
  - `-T` — показать символы табуляции как “^|”; 
  - `-E` — показать символы конца строки как “$”.
- `head -n [i] [имя файла]` отобразить десять первых строк
- `tail -n [i] [имя файла]` отобразить десять последних строк
- `more [имя файла]` просмотреть большой файл (Enter или пробел, чтобы перейти вперед; для поиска текста / (косая черта); клавиша Q, чтобы выйти)
- `less [имя файла]` работает быстрее, не загружает сразу весь файл (перемещение с помощью Page Up и Page Down)
- `sort [имя файла] > [отсортированный файл]` отсортировать текстовый файл
- `od <опции> <название_файла>` : просматривает бинарные файлы в виде дампа байтов
- `tee` : берёт вывод и записывает в файл. Пример: `echo "last string" | sudo tee -a file` — добавит последнюю строку в файл.
- `sort [имя файла] > [отсортированный файл]` отсортировать текстовый файл
- `uniq [имя файла] > [имя файла без дублей]` удалить дубликаты в текстовом файле
- `nano` текстовый редактор. Синтаксис: `nano <название_файла>`

## Поиск файлов и внутри них
- `locate [имя файла]` быстро найти файл (перед вводом обновить `updatedb`)
- `which [имя приложения]` найти путь к файлу 
- `find [параметры]` команда для поиска файлов с более гибкими параметрами 
- `grep [опции] [паттерн] [имя файла]` команда фильтрации
  - `-i` — игнорировать регистр и включать все прописные/строчные буквы;
  - `-r` — выполнить рекурсивный поиск во вложенных каталогах;
  - `-l` — распечатать имена файлов, в которых найден текст, соответствующий фильтру
- `awk /[критерий поиска]/ [опции] [имя файла]` расширенный инструмент для фильтрации 
- `cut` извлечение подстрок (например, cut -d ":" -f 2 извлечение пароля)

## Сжатие файлов
Есть несколько способов (алгоритмов) для сжатия файлов:
- **Архив Tar**
- `tar cf compressed.tar files` Сжать с использованием расширения tar
- `tar xf compressed.tar` Извлечь файл, сжатый tar
  - `-c` — создать архив; 
  - `-r` — добавить файлы в архив; 
  - `-A` — добавить содержимое tar-файлов в архив; 
  - `--delete` — удалить файлы из архива (невозможно на архивных лентах); 
  - `-t` — вывести список файлов в архиве; 
  - `-x` — извлечь файлы из архива; 
  - `-f` — информация будет извлекаться из файла; 
  - `-v` — расширенный вывод информации о выполняемых действиях.
- **Архив Gz**
- `tar cfz compressed.tar.gz files` Создать compressed.tar.gz из файлов
- `tar xfz compressed.tar.gz` Извлечь compressed.tar.gz
- `gzip file.txt > compressed.txt.gz` Создать файл compressed.txt.gz
- `gzip -d compressed.txt.gz` Извлечь compressed.txt.gz
  - `-d` — декомпрессия файла; 
  - `-t` — проверяет целостность архива; 
  - `-v` — расширенный вывод информации.
- **Архив Bz2**
- `tar cfj compressed.tar.bz2 files` Создать compressed.tar.bz2 из файлов
- `tar xfj compressed.tar.bz2 `Распаковать compressed.tar.bz2
- **ZIP-архив**
- `zip compressed.zip files` Создать compressed.zip из файлов 
- `unzip compressed.zip` Распаковать сжатые файлы .zip

## Управление вводом/выводом
В работе с командной строкой Linux есть понятия стандартных устройств ввода, вывода и вывода ошибок.
- **stdin** - стандартное устройство ввода. Имеет файловый указатель 0. Автоматически открывается всеми процессами.
- **stdout** - стандартное устройство вывода. Имеет файловый указатель 1. Автоматически открывается всеми процессами.
- **stderr** - стандартный поток ошибок (специальное устройство вывода для сообщений об ошибках. Имеет файловый указатель 2. Автоматически открывается всеми процессами).

Операторы перенаправления способны изменять направление вывода и ввода информации. Так оператор:
- `>` - перенаправляет стандартный поток в файл (другой поток). При этом если файл существует, то он перезаписывается, если не существует - создается.
- `>>` - перенаправляет стандартный поток в файл. При этом если файл существует, то информация добавляется в конец, если не существует - файл создается 
- `<` - перенаправляет содержимое указанного файла на стандартный ввод программе.
- `>&` - перенаправляет стандартные потоки вывода и ошибок друг в друга.

Существуют средства группирования команд:
- `cmd1 arg ...; cmd2 arg ...; ... cmdN arg ...` - последовательное выполнение команд;
- `cmd1 arg ...& cmd2 arg ...& ... cmdN arg ...` - асинхронное выполнение команд;
- `cmd1 arg ... && cmd2 arg ...` - зависимость последующей команды от предыдущей таким образом, что последующая команда выполняется, если предыдущая выдала нулевое значение;
- `cmd1 arg ... || cmd2 arg ...` - зависимость последующей команды от предыдущей таким образом, что последующая команда выполняется, если предыдущая выдала ненулевое значение.
